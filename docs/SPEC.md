# atelier. 実装仕様（ツール定義の契約）

atelier. は「AIに渡すプロンプトを組み立てる」6ツールを1画面で切り替えるスタジオ。
スタック: Vite + React 19 + TypeScript + Tailwind v4。UIはスキーマ駆動で自動生成される。

## ツール実装者が書くもの

`src/tools/<id>/index.ts` に `export const def: ToolDef` を1つ。型は `src/lib/types.ts` が正典。
補助データを分けたければ同フォルダに `data.ts` / `prompt.ts` を追加してよい（フォルダ外は触らない）。

- `sections`: ステップ番号つきのフォーム定義（num: "01"〜、badge: required/optional）
- `defaults`: **全フィールドIDのキーを含める**（multi は `[]`、toggle は boolean、repeater は `[]`）
- `build(state)`: プロンプト文字列 + meta（枚数/比率/トーン等のチップ）+ warnings（未入力注意）

`src/lib/prompt.ts` のヘルパー（joinBlocks / block / bullets / splitLines）を使うと揃う。

## フィールド種別の使い分け

- `segment`: 2〜5択の排他（例: 文体、モード）
- `pills`: 6択以上の排他。**もう一度押すと解除**される仕様（任意項目向き）
- `multi`: 複数選択
- `cards`: スタイルプリセット等、説明文とタグを見せたい大きな選択肢（columns: 2 or 3）
- `repeater`: スライド別・画像別・セクション別の設定。`countField` に number フィールドIDを渡す
- `showIf`: 条件表示。**repeater 内の showIf はそのアイテムの状態を受け取る**

## プロンプト品質バー（画像生成系ツール共通）

hituji.blog 系ツールの出力を研究した結果、効くプロンプトの構造は:

1. **【出力形式】** 枚数・比率・「1回の画像生成=1枚。N枚をグリッドにまとめるの絶対禁止」を最初に明言
2. **【ビジュアルスタイル】** 基調色（HEX指定で厳密に）・アクセント1色・質感・禁止する質感（グラデ/影/3D等）を1段落で
3. **【共通ルール】** [デザインルール]（世界観統一・構図のバリエーション・1枚1メッセージ・安全余白=短辺7〜8%・余白率30%以上・欧文キャッチの添え方）と [禁止事項]（AIテンプレ=白角丸カード横並び禁止・①②③矢印ステップフロー禁止・テキスト詰め込み禁止）
4. **役割別レイアウト指定**（表紙/本編/まとめ等で描き分け、それぞれに禁止事項）
5. **構成リスト**（1枚ずつ役割と内容）

この骨格は踏襲しつつ、**文言はすべて自分の言葉で書き下ろす**こと（下記オリジナリティ原則）。

## オリジナリティ原則（重要）

- これは hituji.blog ツールの複製ではなく**ゆうきくん独自のツール**。プリセット名・説明文・プロンプト本文は**必ずオリジナルの文章で書き下ろす**。参考資料（originals.md）の文言をそのまま転記しない
- プリセットの体系（どんなスタイル軸を持つか）は一般的なデザイン語彙（SaaS風・和モダン・エディトリアル等）を使ってよいが、ラインナップは自分で設計し直す
- ゆうきくんはUI/UXデザイナー。プリセットの説明文は**デザイナーが読んで信頼できる具体性**で（「おしゃれ」等の空語禁止）

## UIライティング

- 日本語。ですます調は使わず簡潔体（「〜を選ぶ」「空欄なら自動」）
- **装飾絵文字は一切使わない**（ゆうきくんの絶対ルール）
- help テキストは1行で実用的に（迷いどころだけ补足）
