# 半自動HP制作ワークフロー設計

atelier. × HP制作ファクトリー（`~/Desktop/Project/HP/`）× Claude Code / Codex を束ねて、
「ヒアリングから公開まで、人間はデザイン判断だけやる」状態をつくるための設計書。

## 全体像

```
[A 営業リサーチ]──[B ヒアリング]──[C トーン選定]──[D ビジュアル合意]──[E 実装]──[F 公開]──[G 運用]
   HP/sales-       HP/hearing       atelier.web      atelier.web        Claude     HP/       HP/seo
   research        (hearing.md      のトーン8種      モック画像         Code +     deploy
                    = SSOT)         = HP/tones と    プロンプト         スペックMD Cloudflare
                                    同一体系                            + coding
```

ポイント: **atelier.web のトーン8種は HP/design-system/tones/ と同じHEX・同じ体系**。
そのため atelier で作ったモックとスペックが、そのまま HPファクトリーの実装フェーズに接続する。

## 1案件の標準ループ（目標: 実働1〜2日）

1. **ヒアリング**（人間30分）: HP/hearing スキルで `hearing.md`（SSOT）を作る
2. **atelier 入力**（人間5分）: hearing.md を見ながら web ツールにサイト種別・強み・CTA・トーンを入力
3. **モック画像生成**（AI 10分）: FV+主要セクションのプロンプトを ChatGPT Images に貼る。
   トーン違い2〜3案を並べ、**顧客（or 自分）は画像を選ぶだけ**
   → 従来のグレースケールWFより「完成の絵」で合意が取れるので、後工程の手戻りが減る
4. **スペックMD生成**（AI 1分）: 同じ入力から実装スペックMDを出力。選ばれたモック画像とセットで正典化
5. **実装**（AI 数時間・人間は指示のみ）: Claude Code にスペックMD＋モック画像を渡し、
   HP/coding スキル（Astro + Tailwind v4・トークン経由・1セクション1コンポーネント）で実装
   - 単純な量産作業（残りセクションのコンポーネント化・レスポンシブ調整・画像最適化）は Codex に並列で投げる
   - 検証: Lighthouse 90+、AIっぽさ回避チェックリスト（DESIGN.md §3）を通す
6. **微調整ループ**（人間=デザイナー判断、AI=手を動かす）: 下記「微調整エディタ」参照
7. **公開**: HP/deploy スキル（プレビューURL顧客確認 → Cloudflare Pages 本番）

## 微調整エディタ構想（実装とリンクするデザイン修正）

段階的に3ステップで「最強」に近づける:

### Step 1（今すぐできる・道具は揃ってる）
- design-mock スキル方式: **調整パネル付きプレビューHTML**。トークン（色・余白・角丸・書体）を
  スライダー/ピッカーで動かし、「CSS変数で書き出し」→ Claude Code が `src/styles/tokens.css` に還流
- 決定は必ずトークンファイルに戻す（HP憲法の「HTMLで見せて、正典はMDに還流」原則そのまま）

### Step 2（次のスプリント）
- Astro サイトに **dev専用トークンインスペクタ** を組み込む:
  `?tuner=1` で右下にパネルが出て、実サイト上で `--c-primary` 等をライブ編集 → JSONダウンロード
  → 「この JSON 適用して」の一言で Claude Code がトークンを更新
- セクション単位の差し替え: 1セクション1コンポーネントなので「heroだけ案Bに」が1ファイル修正で済む

### Step 3（将来・エージェント連携）
- Claude Code + ブラウザMCP（Preview / Chrome）で「実画面を見ながら指示 → 該当コンポーネント特定 → 修正 → スクショで確認」の全自動ループ
- ゆうきくんは Figma 感覚で「ここ8px詰めて」「この見出し明朝に」と話すだけ

## 役割分担

| プレイヤー | 担当 |
|---|---|
| atelier. | 入力の標準化・プロンプト/スペックの品質固定（属人性をゼロに） |
| ChatGPT Images / Nanobanana | モック画像・素材画像の生成 |
| Claude Code | 実装・統合・検証・デプロイ（判断が要る作業すべて） |
| Codex | 並列の量産作業（コンポーネント複製・機械的な調整） |
| ゆうきくん | デザイン最終判断・顧客コミュニケーション（ここは自動化しない） |
